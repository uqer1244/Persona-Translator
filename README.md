# PersonaASMR-Translator

로컬 Apple Silicon GPU 환경에 최적화된 로컬 VLM (Gemma 4 12B) 기반의 **상황극 및 ASMR 대본 전문 맞춤형 번역 시스템**입니다. 

ASMR 콘텐츠 특유의 섬세한 말투(페르소나)와 작품 배경 상황을 로컬 LLM이 입체적으로 파악하고, 대사 이외의 각종 연출 지시문 및 시간 정보(SRT 타임코드)를 깨뜨리지 않으면서 자연스러운 한국어로 실시간 번역해 줍니다.

---

## 주요 기능

### 1. Apple Silicon GPU 로컬 가속 (MLX VLM)
- Apple Silicon 환경에서 최상의 속도와 효율을 발휘하는 `mlx-vlm` 라이브러리를 탑재했습니다.
- 외부 API 키가 불필요하며, 대용량 대본도 로컬 오프라인 환경에서 유출 우려 없이 안전하고 신속하게 번역합니다.
- `TurboQuant (KV Cache 3.5-bit)` 및 메모리 비우기 자동화 설정을 지원하며, 16GB RAM 사양의 맥(M3 등) 환경에서는 VRAM 부족(OOM) 방지를 위해 4비트 양자화 모델(`Gemma4_12B_4bit_mlx`) 로드가 기본 설정되어 안전하게 동작합니다.

### 2. 다중 대본 파일 업로드 및 자동/수동 불러오기
- **자연스러운 트랙 정렬 (Natural Sort)**: 업로드하거나 로컬에서 스캔한 트랙 파일들(예: `track1.txt` ~ `track10.txt`)이 단순 사전식 정렬(1 -> 10 -> 2) 대신 실제 트랙 숫자 크기 순서(`track1` -> `track2` -> `track10`)대로 올바르게 정렬 및 로딩됩니다.
- **깨짐 없는 인코딩 감지기 (Robust Text Decoder)**: Shift_JIS/CP932, CP949, UTF-16 등 다양한 대본 인코딩 형식을 순차적으로 시도해 자동 디코딩하여 문자 깨짐(Mojibake) 현상을 원천 차단합니다.
- **백엔드 파일 및 백업 자동 제외**: 대본 로컬 스캔 시 번역이 완료된 파일(파일명 끝이 `_translated`인 경우)이나 `translation_backup`, `temp_backups` 등 백엔드 작업 디렉토리는 자동으로 제외되어 원문 파일만 깔끔하게 불러옵니다.
- **SRT 자막 동적 청커**: 자막 번호 및 타임라인(`00:00:01,000 --> 00:00:04,000`)의 포맷 손상 없이 대사만 블록 단위로 번역합니다. 글자 수 기준(기본 800자)으로 자막 블록들을 **동적으로 묶어 청킹**함으로써 번역 속도를 4배 향상하고 VRAM 부하를 최소화했습니다.

### 3. 작품 소개 이미지 다중 분석 & 다중 화자 페르소나 추출
- 개수 제한 없이 ASMR 작품의 홍보 일러스트, DLsite 트랙 디테일 컷 등을 업로드해 VLM으로 연계 분석할 수 있습니다.
- **다중 화자 어조 수집**: 대본 내에 주요 화자가 2명 이상 등장하는 경우(예: 히로인 A와 히로인 B 등), 캐릭터 분석 프롬프트가 이를 감지하여 각각의 어조 및 관계 설정을 누락 없이 수집하고 페르소나 정보에 기록합니다.

### 4. 장면 경계 인지형 (Scene-aware) 스마트 청킹 및 글로벌 맥락 주입
- **장면 경계 분할**: 글자 수 한계뿐만 아니라 대본 내의 사운드 이펙트 지시문(`[SE: ...]`, `[BGM: ...]`), 트랙 구분선, 자막 타임스탬프 등 장면 전환부(Boundary)를 자동 감지하여 맥락이 깨지지 않게 스마트하게 청크를 쪼갭니다.
- **글로벌 배경 상황 반영**: 페르소나 도출 시 전체 스토리 배경 상황을 자동으로 함께 요약하여 저장하며, 번역 프롬프트에 지속 주입하여 대본이 길어져도 일관된 고품질 번역을 보장합니다.

### 5. 페르소나 자동 저장 및 라이브러리 공유
- 화자와 청자의 관계 설정, 말투 특징, 번역 규칙 및 배경 상황 설정을 상세히 저장 및 수동 편집할 수 있습니다.
- **실시간 자동 저장(Autosave)**: 인풋박스나 용어집 에디터를 편집하면 백업 폴더 내 `persona.json` 파일에 실시간 보존되며, 재시작 시 자동으로 복원됩니다.
- **페르소나 공유 라이브러리**: 다른 프로젝트에서 도출했던 페르소나 설정을 메뉴에서 선택해 원클릭으로 이식할 수 있습니다.

### 6. 번역 사고 단계 출력 및 최종 대본 파서 연동
- **구조적 요소 분류 가이드라인**: VLM이 문맥에 쓸려 줄글이나 소설 형태로 대본을 번역하는 현상을 방지하고자, 번역 전 **`[대본 분석]`** 단계(대사/상황어/효과음/타임스탬프 분류)를 출력한 뒤 **`[최종 번역 결과]`**를 내보내도록 유도합니다.
- **번역문 자동 파서**: 실시간 스트리밍 시에는 이 상세한 분석 과정을 보여주어 동작 상태를 확인하게 하고, 최종 누적 결과 및 파일 저장 시에는 `extract_final_translation` 파서가 번역 대본 본문만 자동으로 추출하여 깨끗한 원문 대본 포맷으로 저장합니다.

### 7. 실시간 대본 채색 (하이라이팅) 및 옵션 선택 토글
- 실시간으로 토큰을 생성할 때마다 Streamlit 세션을 100ms 주기로 새로고침하여 빠른 스트리밍을 제공합니다.
- **정밀 전각/반각 기호 채색**: 전각 대괄호(`［ ］`), 전각 소괄호(`（ ）`), 전각 별표(`＊ ＊`) 및 전각 콜론(`：`) 등 동양권 대본에 사용되는 모든 기호를 정상 감지하여 파스텔 톤 하이라이트 채색을 입힙니다.
- **화자별 고유 색상(Coloring) 매핑**: 낫표(`「`, `『`) 또는 콜론 기호 앞의 화자 이름을 감지하여 인물별로 각기 다른 파스텔 톤 고유 색상을 실시간 부여합니다.
- **채색 토글 활성화 옵션**: 사이드바의 체크박스 하나로 실시간 채색 스팬 코드 적용 여부를 즉각 키고 끌 수 있어 가독성 우선 환경을 선택할 수 있습니다.

### 8. 무한 루프 감지 및 우회 설정 자동 재시도 (Auto-Retry)
- 로컬 경량화 VLM 기동 시 발생하기 쉬운 **무한 반복 루프 현상**을 감지(`has_repetition`)하여 즉시 차단합니다.
- 루프 발생 시 온도를 상승시키고 반복 방지 패널티를 강화하여 **최대 3회 자동 재시도**를 거침으로써 오류 구간을 자가 치유(Self-healing)합니다.

### 9. 말투 및 포맷 자동 교정 (Refiner)
- 번역이 끝난 결과물에 대해 2차 정밀 교정을 진행하여 말투가 뒤섞이는 현상(반말/존댓말 혼용)이나 괄호 쌍 불일치 포맷 오류를 자동으로 완벽하게 다듬어 줍니다.

## 시작 가이드

### 1. 가상환경 및 의존성 설치
터미널을 열고 프로젝트 폴더 내에서 다음 명령어를 차례로 입력하여 Python 가상환경을 구축하고 라이브러리를 설치합니다. (Python 3.11 권장)

```bash
# 가상환경 생성 및 활성화
python -m venv .venv
source .venv/bin/activate

# 의존성 패키지 설치
pip install -r requirements.txt
```

### 2. 로컬 모델 준비
다운로드받은 MLX 포맷 모델 폴더(예: `Gemma4_12B_4bit_mlx` 등)를 프로젝트 하위의 `models/` 디렉토리에 저장합니다.
```
ASMR_ADV/
├── models/
│   └── Gemma4_12B_4bit_mlx/  <-- 다운로드한 모델 폴더
│       ├── config.json
│       ├── model.safetensors (또는 .npz 가중치 파일들)
│       └── tokenizer.json
```
- *참고: 사이드바에서 "직접 경로 입력..."을 선택하면 Mac OS 내 다른 경로에 있는 모델 폴더도 직접 절대경로로 기입하여 로드할 수 있습니다.*

### 3. 애플리케이션 실행
터미널에서 아래의 명령어를 입력하여 Streamlit 로컬 서버를 실행합니다.
```bash
streamlit run app.py
```
- 구동이 완료되면 자동으로 웹 브라우저(`http://localhost:8501`)가 열리며 번역 대시보드를 사용할 수 있습니다.

## 기술 스택
- **Core Engine**: `mlx-vlm` (Gemma 4 Unified VLM 아키텍처 연동)
- **Framework**: `Streamlit` (UI 및 백엔드 스레딩 구성)
- **Libraries**: `pypdf`, `pandas`, `transformers`, `huggingface_hub`
