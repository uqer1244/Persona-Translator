# PersonaASMR-Translator

로컬 Apple Silicon GPU 환경에 최적화된 로컬 VLM (Gemma 4 12B) 기반의 **상황극 및 ASMR 대본 전문 맞춤형 번역 시스템**입니다. 

ASMR 콘텐츠 특유의 섬세한 말투(페르소나)와 작품 배경 상황을 로컬 LLM이 입체적으로 파악하고, 대사 이외의 각종 연출 지시문 및 시간 정보(SRT 타임코드)를 깨뜨리지 않으면서 자연스러운 한국어로 실시간 번역해 줍니다.

---

## 주요 기능

### 1. Apple Silicon GPU 로컬 가속 (MLX VLM)
- Apple Silicon 환경에서 최상의 속도와 효율을 발휘하는 `mlx-vlm` 라이브러리를 탑재했습니다.
- 외부 API 키가 불필요하며, 대용량 대본도 로컬 오프라인 환경에서 유출 우려 없이 안전하고 신속하게 번역합니다.
- `TurboQuant (KV Cache 3.5-bit)` 및 메모리 비우기 자동화 설정을 통해 16GB RAM 사양의 맥에서도 VRAM 부족(OOM) 현상 없이 구동됩니다.

### 2. 다중 대본 파일 업로드 및 병합 기능
- **텍스트 입력**: 대본 원문을 웹 화면에 직접 복사-붙여넣기하여 즉시 번역 가능.
- **다중 파일 일괄 업로드**: 여러 개의 대본 파일(`.txt`, `.srt`, `.pdf`)을 동시에 드롭해 업로드할 수 있습니다. 
- **수동 파싱 및 병합**: `[업로드된 대본 파일 파싱 및 불러오기]` 버튼을 따로 제공하여 사용자가 원할 때 파싱을 가동합니다. 파일들은 알파벳 순서대로 자동 정렬되어 병합되며, 다중 일반 텍스트 병합 시 트랙 구분선(`[Track 1 (파일명)]`)이 자동으로 생성되어 장면 흐름 파악을 돕습니다.
- **SRT 자막 동적 청커**: 자막 번호 및 타임라인(`00:00:01,000 --> 00:00:04,000`)의 포맷 손상 없이 대사만 블록 단위로 번역합니다. 기존의 고정 5블록 방식 대신 글자 수 기준(기본 800자)으로 자막 블록들을 **동적으로 묶어 청킹**함으로써 번역 속도를 4배 향상하고 VRAM 부하를 최소화했습니다.

### 3. 작품 소개 이미지 다중 분석 (VLM 멀티모달)
- 개수 제한 없이 ASMR 작품의 홍보 일러스트, DLsite 트랙 디테일 컷 등을 함께 업로드할 수 있습니다.
- Gemma 4 VLM 모델이 시각 일러스트 이미지와 텍스트를 연계 분석하여 캐릭터의 성격, 외모 분위기, 표정 등을 어투(페르소나) 설정에 정밀하게 반영합니다. 이미지 분석 마크다운은 백업 경로에 `.analysis.txt` 파일로 개별 보존됩니다.

### 4. 장면 경계 인지형 (Scene-aware) 스마트 청킹 및 글로벌 맥락 주입
- **장면 경계 분할**: 글자 수 한계뿐만 아니라 대본 내의 사운드 이펙트 지시문(`[SE: ...]`, `[BGM: ...]`), 트랙 구분선, 자막 타임스탬프 등 장면 전환부(Boundary)를 자동 감지하여 맥락이 깨지지 않게 스마트하게 청크를 쪼갭니다.
- **글로벌 배경 상황 반영**: 페르소나 도출 시 전체 스토리 배경 상황을 자동으로 함께 요약하여 저장합니다. 번역 엔진이 작동할 때 모든 번역 청크 프롬프트에 이 **배경 상황 맥락(Global Context)**을 상시 주입하여 대본이 길어져도 일관된 고품질 번역을 보장합니다.

### 5. 페르소나 자동 저장 및 라이브러리 공유
- 화자와 청자의 관계 설정, 말투 특징, 번역 규칙 및 배경 상황 설정을 상세히 저장 및 수동 편집할 수 있습니다.
- **실시간 자동 저장(Autosave)**: 페르소나 설정창의 인풋박스나 용어집 에디터를 편집하면 해당 프로젝트 백업 폴더 내 `persona.json` 파일에 실시간 보존되며, 재시작 시 자동으로 복원됩니다.
- **페르소나 공유 라이브러리**: 다른 프로젝트에서 도출했던 페르소나 설정을 드롭다운 메뉴에서 선택해 원클릭으로 현재 번역 창에 이식할 수 있어, 시리즈물이나 동일 성우/캐릭터 작품의 번역 연속성을 완벽히 유지합니다.

### 6. 실시간 대본 채색 및 초고속 스트리밍 뷰어
- 백그라운드 번역 스레드가 토큰을 생성할 때마다 **Streamlit 세션을 직접 새로고침(Rerun Trigger, 100ms 주기)**하여 지연 없이 실시간 글자 단위 스트리밍을 구현했습니다.
- 번역 과정은 전체 텍스트 뷰어 및 개별 청크 목록 상에서도 실시간으로 흐르듯 출력되며, 종류별로 하이라이트 채색을 입힙니다.
  - `[속삭임]` 등 대괄호 연출 지시문: **굵은 파스텔 오렌지색**
  - `(한숨)` 등 소괄호 감정 지시문: *기울임 파스텔 핑크색*
  - `*소곤소곤*` 효과음 지시문: *기울임 파스텔 민트색*
  - SRT 자막 타임라인: 흐린 monospace 회색

### 7. 무한 루프 감지 및 우회 설정 자동 재시도 (Auto-Retry)
- 로컬 경량화 VLM 기동 시 발생하기 쉬운 **무한 반복 루프 현상(예: 동일 어구 무한 반복 출력)**을 실시간으로 감지(`has_repetition`)하여 즉시 차단합니다.
- 루프 발생 시 온도를 상승(`temp + 0.15`)시키고 반복 방지 패널티를 강화(`penalty + 0.1`)하여 **최대 3회 자동 재시도**를 거침으로써 오류 구간을 슬기롭게 탈출(Self-healing)합니다.

### 8. 말투 및 포맷 자동 교정 (Refiner)
- 번역이 끝난 결과물에 대해 2차 정밀 교정을 진행하여 말투가 뒤섞이는 현상(반말/존댓말 혼용)이나 괄호 쌍 불일치 포맷 오류를 자동으로 완벽하게 다듬어 줍니다.

## 시작 가이드

### 1. 가상환경 및 의존성 설치
터미널을 열고 프로젝트 폴더 내에서 다음 명령어를 차례로 입력하여 Python 가상환경을 구축하고 라이브러리를 설치합니다. (Python 3.11 권장)

```bash
# 가상환경 생성 및 활성화
python -m venv .venv
source .venv/bin/activate

# 의존성 패키지 설치
pip install -r requirements.txt
```

### 2. 로컬 모델 준비
다운로드받은 MLX 포맷 모델 폴더(예: `Gemma4_12B_4bit_mlx` 등)를 프로젝트 하위의 `models/` 디렉토리에 저장합니다.
```
ASMR_ADV/
├── models/
│   └── Gemma4_12B_4bit_mlx/  <-- 다운로드한 모델 폴더
│       ├── config.json
│       ├── model.safetensors (또는 .npz 가중치 파일들)
│       └── tokenizer.json
```
- *참고: 사이드바에서 "직접 경로 입력..."을 선택하면 Mac OS 내 다른 경로에 있는 모델 폴더도 직접 절대경로로 기입하여 로드할 수 있습니다.*

### 3. 애플리케이션 실행
터미널에서 아래의 명령어를 입력하여 Streamlit 로컬 서버를 실행합니다.
```bash
streamlit run app.py
```
- 구동이 완료되면 자동으로 웹 브라우저(`http://localhost:8501`)가 열리며 번역 대시보드를 사용할 수 있습니다.

---

## 기술 스택
- **Core Engine**: `mlx-vlm` (Gemma 4 Unified VLM 아키텍처 연동)
- **Framework**: `Streamlit` (UI 및 백엔드 스레딩 구성)
- **Libraries**: `pypdf`, `pandas`, `transformers`, `huggingface_hub`
